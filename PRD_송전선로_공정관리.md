# PRD — 송전선로 지중화 건설공사 공정관리 웹앱

| 항목 | 내용 |
|---|---|
| 문서 버전 | v1.0 (확정) |
| 작성일 | 2026-06-11 |
| 목적 | 송전선로 지중화 공사(터파기~본포장)의 구간별·공정별 진행현황을 매일 입력하고 한눈에 보는 대시보드 |
| 사용 형태 | PC 입력·조회·인쇄 중심, 모바일 조회 보조 / 인터넷 연결 시 어디서나 사용 |

---

## 1. 배경 및 문제 정의

- 공구(작업조)가 여러 구간을 **이리저리 옮겨다니며** 공사를 진행하기 때문에, 전체 공사가 지금 얼마나 진척됐는지 한눈에 파악하기 어렵다.
- 각 위치는 **7단계 사이클**(터파기 → ELP관 매설 → 차폐판 설치 → 되메우기 → 표층 메우기 → 가포장 → 본포장)을 순서대로 거친다.
- 매일 약 20m씩 진행되며, 현장은 **체인 NO.**(0부터 20m 간격)로 위치를 관리한다.
- **해결**: 매일 값을 입력하면 구간별·공정별 진행이 자동으로 도식화되어, "어디가 어느 단계까지 갔는지"와 "공정별 총 진행량"을 즉시 확인.

## 2. 대상 사용자

| 사용자 | 사용 환경 | 주요 행동 |
|---|---|---|
| 현장 공정관리 담당(주 사용자) | PC | 매일 진행값 입력, 대시보드 조회, 인쇄 |
| 관리자/상급자 | PC, 가끔 모바일 | 전체 현황 조회, 인쇄물 확인 |
| 현장 인원 | 모바일 | 간헐적 조회 |

## 3. 공사 구조 (기준 데이터)

### 3.1 구간 정의

전체 연장 **3,448m**, 체인 NO.는 0부터 20m 간격(예: NO.16 = 320m).

| 구간 | 지역 | 시점(m) | 종점(m) | 연장(m) | 체인 NO. | 공법 |
|---|---|---|---|---|---|---|
| 1구간 | 수원 | 0 | 320 | 320 | 0 ~ 16 | 개착식 |
| 2구간 | 수원 | 320 | 740 | 420 | 16 ~ 37 | 개착식 |
| 3구간 | 수원 | 740 | 1,620 | 880 | 37 ~ 81 | 개착식 |
| 4구간 | 수원 | 1,620 | 2,320 | 700 | 81 ~ 116 | 개착식 |
| 5구간 | 용인 | 2,320 | 2,983 | 663 | 116 ~ 149.15 | 개착식 |
| 6구간 | 용인 | 2,983 | 3,232 | 249 | 149.15 ~ 161.6 | **혼합**(아래) |
| 7구간 | 용인 | 3,232 | 3,448 | 216 | 161.6 ~ 172.4 | 개착식 |

- **수원(1~4구간)**: 2,320m
- **용인(5~7구간)**: 1,128m

> 체인 NO. 소수 표시 처리는 구현 시 결정(반올림 표기 + 실측 m 병기 권장). 시점/종점 m는 위 표가 정답값.

### 3.2 6구간 공법 분리 (특수)

6구간(249m)은 공법이 셋으로 나뉘고, **개착식만 7단계 사이클을 적용**한다.

| 세부 공법 | 연장(m) | 진행 관리 방식 |
|---|---|---|
| 강관압입 | 48 | 별도 공정(추진 진행 m / 48m) |
| 지향성압입 | 179 | 별도 공정(추진 진행 m / 179m) |
| 개착식 | 22 | 표준 7단계 사이클 |

- 압입(강관·지향성) 구간 합 **227m**는 7단계와 무관한 **별도 단순 진행 공정**으로 관리(진행 m / 완료 m).
- 따라서 **개착식 7단계 적용 총 연장 = 3,448 − 227 = 3,221m**.

### 3.3 7단계 공정 사이클 (개착식)

순서대로(계단식) 진행되며, 각 위치는 앞 단계를 마쳐야 다음 단계로 넘어간다.

| 순번 | 공정 | 비고 |
|---|---|---|
| 1 | 터파기 | |
| 2 | ELP관 매설 | |
| 3 | 차폐판 설치 | |
| 4 | 되메우기 | |
| 5 | 표층 메우기 | |
| 6 | 가포장 | |
| 7 | 본포장 | **= 완료** |

> **계단식 제약**: 임의 구간에서 누적 진행량은 항상  
> `터파기 ≥ ELP매설 ≥ 차폐판 ≥ 되메우기 ≥ 표층 ≥ 가포장 ≥ 본포장`  
> 입력 시 이 불변식을 검증한다(상위 단계가 하위 단계 누적을 넘지 못하게).

## 4. 핵심 기능 요구사항

### 4.1 데이터 입력 (Input)

- **입력 단위**: 구간 × 공정 단계 기준.
- **입력 방식(기본)**: 해당 날짜에 "오늘 진행한 m"를 입력 → 시스템이 누적 합산. (누적값 직접 수정도 가능하게 보조 제공)
- **일자별 기록**: 날짜를 키로 일일 진행 로그 저장 → 추세/히스토리 확보.
- **검증 규칙**:
  - 계단식 불변식 위반 시 경고(예: ELP매설 누적이 터파기 누적 초과 불가).
  - 구간 연장 초과 입력 방지(누적이 구간 연장 초과 불가).
- **6구간 압입**: 강관압입·지향성압입 각각 진행 m 입력(7단계와 분리된 입력 폼).
- **입력 편의**: 자주 쓰는 "오늘 작업한 구간/공정"만 빠르게 입력하는 간편 모드.

### 4.2 시각화 / 대시보드 (Output)

1. **전체 종합 진행률 (헤더 KPI)**
   - 종합 진행률(가중) + 실질 완료율(본포장 m / 전체 연장) 동시 표시.
   - 수원 / 용인 그룹별 진행률 분리 표시.
2. **구간별 진행 현황 (계단식 그래프)**
   - 구간마다 7단계가 어디까지 왔는지 계단(스택/스텝) 막대로 표시.
   - **체인 NO. 전체 표기**: 0~172.4 구간의 모든 체인 NO.를 축에 눈금으로 표시(20m 간격 전부). 시점/종점 라벨 병기.
   - "현재 작업 위치"(가장 앞선 미완료 단계의 선단) 강조.
3. **공정별 누적 합계 (집계 카드/바)**
   - 예: `터파기 총 OOOm / 3,221m`, `가포장 총 OOOm`, `본포장(완료) 총 OOOm` …
   - 7단계 각각 누적 m + 진행률(%).
4. **현재 상태 보드**
   - 각 공구/구간이 지금 어느 공정·어느 위치(체인 NO.)에 있는지.
   - 압입 구간(6구간) 추진 현황 별도 카드.
5. **노선도(지도) 현황 — 신규**
   - 사용자가 올린 노선 지도 이미지 위에 구간/위치별 공정 현황을 색으로 표시(아래 4.5 참조).
6. **추세(선택, 후순위)**
   - 일자별 진행량 추이, 일평균 약 20m 대비 실적.

### 4.3 인쇄

- PC에서 대시보드를 **A3(가로 권장) 인쇄용 레이아웃**으로 출력. 보고용으로 깔끔하게(헤더에 날짜·전체 진행률 요약).
- 전체 체인 NO.가 한 장에 다 들어가도록 A3 기준으로 그래프 폭/눈금 밀도를 조정.

### 4.4 반응형 / 모바일

- PC 우선, 모바일에서는 **조회 중심**의 단순 레이아웃으로 자동 전환.

### 4.5 노선도(지도) 위 위치별 현황 — 신규

사용자가 노선 지도 이미지를 업로드하면, 그 위에 각 위치의 공정 현황을 표시한다.

- **지도 등록**: 이미지(PNG/JPG) 또는 PDF 업로드. (배경으로 고정)
- **노선 보정(캘리브레이션, 1회 설정)**: 지도 위에서 노선 경로를 따라 점을 찍고, 시점(0m)·종점(3,448m)을 지정 → m/체인 NO.가 지도 좌표에 매핑됨.
  - 곡선 노선이면 경유점(vertex)을 여러 개 찍어 폴리라인으로 정의.
- **현황 오버레이**:
  - 노선을 공정 단계별 색으로 칠해 표시(예: 본포장=완료색, 터파기=착수색, 미착수=회색).
  - 또는 구간/체인 NO. 단위로 색 구분.
  - 현재 작업 위치(선단) 마커, 압입 구간(6구간) 별도 표기.
- **인터랙션**: 지도의 한 지점을 클릭하면 해당 위치의 구간·체인 NO.·공정 상태 팝업.
- **인쇄**: 노선도 현황도 A3 인쇄 대상에 포함.

> 구현 메모: 업로드 이미지는 GPS 좌표가 없는 그림이므로, 정밀 측위가 아니라 **이미지 좌표 ↔ 노선 진행거리(m)** 매핑 방식으로 처리. 한 번 보정해두면 이후엔 입력값에 따라 자동으로 색이 갱신됨.

## 5. 진행률 계산 정의

- **공정별 진행률** = 해당 공정 누적 m / 개착식 적용 총 연장(3,221m).
- **구간 종합 진행률(가중)** = Σ(공정별 누적 m × 가중치) / (구간 연장 × Σ가중치).
  - 기본 가중치는 7단계 **균등(각 1/7)**, 추후 단계별 비중 조정 가능하게 설계(설정값).
- **실질 완료율** = 본포장 누적 m / 전체 연장(3,448m, 압입 완료 포함).
- **압입 구간 진행률** = 추진 완료 m / 압입 연장(227m).

> 균등 가중을 기본으로 하되, 실제 공정 비중(예: 본포장 가중↑)을 반영하고 싶으면 설정에서 조정. 헤드라인은 "실질 완료율(본포장)"과 "종합 진행률"을 함께 보여 오해를 줄인다.

## 6. 데이터 모델 (초안)

```
Section (구간)         : id, name, region(수원/용인), start_m, end_m, length_m, method
SectionMethod (6구간)  : section_id, method(강관압입/지향성압입/개착식), length_m
ProcessStage          : id, order(1~7), name
DailyEntry (일일입력)  : date, section_id, stage_id, delta_m            // 오늘 진행 m
Cumulative (파생)      : section_id, stage_id, cumulative_m             // 합산 결과
JackingEntry (압입)    : date, section_id, method, delta_m              // 압입 진행
Settings              : stage_weights[], chain_interval(=20)
```

- 누적값은 DailyEntry 합산으로 산출(원장은 일일 로그). 보정용 누적 직접 입력은 조정(adjustment) 레코드로 처리.

## 7. 기술 / 배포 (권장안)

> "인터넷에 연결만 되어 있으면 어디서나 입력·조회" 요구 → **호스팅형 웹앱 + 클라우드 DB**.

- **프론트엔드**: React(Vite) 또는 Next.js + 차트 라이브러리(Recharts/Chart.js).
- **백엔드/DB**: **Supabase(PostgreSQL)** 권장 — 무료 티어로 시작 가능, 인증·API·자동 백업 내장.
- **배포**: Vercel(프론트) + Supabase(데이터). 도메인/로그인 부여.
- **접근 제어**: 간단한 비밀번호/로그인(소수 인원). 입력 권한 / 조회 권한 분리(후순위).

### 백업 (사용 중 별도 협의 — 권장안)

- Supabase 자동 일일 백업 + 주기적 **엑셀/JSON 내보내기** 버튼 제공(수동 백업·공유용).
- 초기에는 "엑셀 내보내기" 기능만으로 시작하고, 운영하며 필요 시 자동 백업 정책 강화.

> 참고: 만약 인원이 사실상 1명이고 서버 운영을 피하고 싶다면 브라우저 로컬 저장 + 엑셀 백업도 가능하나, "어디서나 입력/조회" 요구를 충족하려면 위 호스팅형이 적합합니다.

## 8. 범위 (Scope)

**v1 (MVP)**
- 기준 데이터(7구간, 6구간 공법 분리, 7단계) 세팅
- 일일 입력(구간×공정 오늘 진행 m) + 계단식/연장 검증
- 대시보드: 종합 진행률, 구간별 계단 그래프(체인 NO. 전체 표기), 공정별 누적 합계, 현재 상태 보드
- 압입(6구간) 별도 진행 표시
- A3 인쇄 레이아웃 + 모바일 조회 대응
- 엑셀/JSON 내보내기(백업)

**v1.5**
- 노선도(지도) 업로드 + 노선 보정 + 위치별 현황 색 오버레이 (4.5)

**v2 (후순위)**
- 일자별 추세 차트, 일평균 대비 실적
- 단계별 가중치 설정 UI
- 로그인/권한 분리, 첨부(사진) 기록

**범위 외**
- 비용·자재·인력 관리, 공정표(간트)와의 연동, 외부 ERP 연동.

## 9. 미해결 / 추후 결정 사항

1. 백업 정책 강도(자동 백업 주기) — 운영하며 결정.
2. 단계별 가중치(균등 vs 실제 비중) 최종값.
3. 체인 NO. 표기 방식(소수 처리/실측 m 병기).
4. 로그인 필요 수준(공개 링크 + 비밀번호 vs 계정).
5. 압입 구간을 단계 세분(추진 준비/추진/마무리)까지 볼지, 단일 진행 m로 둘지.

**결정 사항**
- 노선도: **지도 이미지 업로드 + 노선 보정** 방식으로 확정(실시간 지도 API 미사용). 한 장의 전체 노선도(시점~종점 모두 포함) 기준.
- 노선 경유지(시점→종점): 지역난방공사 수원지사 북동측 끝(출발) → 영통도서관 → 청명역 → 영통고가 → 중부대로 횡단 → 신동아파밀리에 → 흥덕성당 → 흥덕변전소(도착).
- **노선 형상 보정 규칙**:
  - **영통고가 P턴**: 실제 노선이므로 **그대로 유지**.
  - **SK엔클린 옆 = 중부대로 횡단부 U자 루프**: 자동차 경로상의 우회이므로 **제거하고 지중 남→북 직선 횡단**으로 보정(위쪽 세로 도로로 직결).
  - 그 외 자동차 내비 경로의 회전·우회는 실제 지중 노선 기준으로 직선화.
- 노선도 표시: 구간색 + 공정 현황색 **둘 다(토글)**.
- **대시보드 메인 그래픽**: 사용자가 형태를 최종 선택 예정(후보: 계단식 스택 막대 / 구간별 7단계 개별 바 / 기타). 선택 확정 후 그 형태로 최종 구현.
